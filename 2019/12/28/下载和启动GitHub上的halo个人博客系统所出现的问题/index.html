下载和启动GitHub上的halo个人博客系统所出现的问题

将halo源代码从GitHub上下载过程中出现的问题：

  • 下载速度过慢导致下载失败出现clone failed：

    我尝试过更改hosts地址，然后刷新DNS的方法，失败，这个方法越来越难用了。

    最好的办法就是在码云平台上新建一个仓库，选择导入已有仓库，将GitHub上的项目下载地址直接复制在新建的仓库中，然后创建。接着进入仓库点击克隆下载按钮，这样下载速度就会大大提高。

参考文献：https://www.jianshu.com/p/834e85d1c6b6

项目下载好了后在IDEA中跑不起来出现Lombok报错问题：

方法一：

Mac OS系统在preferences里的plugins搜索Lombok插件下载。并在Build，Execution，Deployment中的Compiler菜单 下的Annotation Processors对Halo项目勾选Enable Annotation Processors，最后点击Apply按钮和OK，重启IDEA。

Windows系统则是在左上角的File中settings中，找到其中的plugins搜索Lombok插件下载，接着就是在Build，Execution，Deployment中的Compiler菜单下的Annotation Processors对halo项目里勾选Enable Annotation Processors，最好点击OK按钮重启IDEA。

方法二：

如若还是报错在项目的build.gradle文件的依赖项dependencies{ }中添加语句：
compile”org.projectlombok:lombok:1.18.8”
annotationProcessor “org.projectlombok:lombok:1.18.8”
